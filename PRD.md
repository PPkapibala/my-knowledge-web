# 🌸 个人知识库网站
## 产品需求说明书（PRD）

**版本：v2.0 | 更新：2026-02-26**

---

## 1. 产品概述

### 1.1 定位

面向个人学习者的轻量级知识收藏与进度管理工具。帮助用户集中管理学习资源链接，追踪学习状态，可视化学习进度。

### 1.2 目标用户

有持续学习习惯、需要管理大量学习链接的个人用户（学生、职场人、技能自学者）。

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 一键收藏 | 粘贴 URL 自动抓取标题，资源集中管理 |
| 完全自定义 | 分类和资源类型均可自由增删改 |
| 状态追踪 | 清晰区分待学习、学习中、已完成 |
| 进度可视 | 学习看板让成就感一目了然 |

## 3. 数据模型

### articles（知识条目）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| title | text | 标题 |
| url | text | 资源链接 |
| description | text | 摘要备注 |
| status | text | 学习状态（见第 5 章） |
| resource_type_id | uuid | 关联资源类型 |
| category_id | uuid | 关联分类 |
| tags | text[] | 标签数组 |
| order | integer | 排序权重 |
| created_at | timestamptz | 创建时间 |
| updated_at | timestamptz | 更新时间 |

### categories（多级分类）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| name | text | 分类名称 |
| parent_id | uuid | 父级分类（null 为顶级） |
| icon | text | Emoji 图标 |
| color | text | 颜色标识 |
| order | integer | 排序权重 |

### resource_types（资源类型）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| name | text | 类型名称 |
| icon | text | Emoji 图标 |
| order | integer | 排序权重 |

### activity_logs（操作日志）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| article_id | uuid | 关联条目 |
| article_title | text | 条目标题快照（防删除后丢失） |
| action | text | 操作类型：`status_change` / `create` / `delete` |
| from_status | text | 变更前状态（status_change 时有值） |
| to_status | text | 变更后状态（status_change 时有值） |
| created_at | timestamptz | 操作时间 |

> 用途：支持学习看板的每周趋势图、学习日历、Streak 统计和最近动态时间轴。

## 4. 功能模块

### 4.1 知识条目管理

**添加条目**
- 粘贴 URL 后自动抓取页面标题和描述（Open Graph / Meta 解析）
- 手动填写：标题、网址、摘要、资源类型、分类、标签
- 资源类型和分类均为必填项

**编辑**
- 双击卡片标题可内联编辑，回车或失焦自动保存

**删除**
- 删除后底部 Toast 提供 5 秒撤销窗口，超时后物理删除

**打开资源**
- 点击卡片跳转到资源原始链接（新标签页）

**排序**
- 卡片支持拖拽排序，顺序持久化至 `articles.order` 字段

---

### 4.2 多级分类系统

- 支持最多 **3 级**分类（系统强制限制）
- 示例：`前端开发` → `React` → `Hooks 专题`

**管理操作**

| 操作 | 说明 |
|------|------|
| 新增 | 填写 Emoji 图标 + 名称 + 颜色，选择父级，即时生效 |
| 重命名 | 双击名称内联编辑，回车/失焦保存 |
| 删除 | 子分类和资源自动上移至父级，不丢失内容 |
| 层级限制 | 超过 3 级时阻止并提示 |

> 分类颜色（`categories.color`）用于左侧导航栏的分类标识条，便于快速区分不同分类。

**入口**：添加资源弹窗的分类字段旁「⚙️ 管理分类」按钮

---

### 4.3 自定义资源类型

用户可自由定义资源维度，不限于预置类型。

**预置类型**：🎬 视频 / 📄 文章 / 📝 笔记 / 📚 课程 / 🔖 其他

**管理操作**

| 操作 | 说明 |
|------|------|
| 新增 | 填写 Emoji + 名称，即时生效 |
| 重命名 | 内联编辑，回车/失焦保存 |
| 删除 | 至少保留 1 个类型；被使用的类型删除后，条目 resource_type_id 置 null |

**入口**：添加资源弹窗的资源类型字段旁「⚙️ 自定义」按钮

---

### 4.4 筛选与搜索

- **分类筛选**：左侧树形导航，点击分类过滤（含子分类）
- **状态筛选**：顶部 Tab 切换
- **类型筛选**：Chips 多选，OR 逻辑
- **关键词搜索**：实时过滤标题、摘要、标签，300ms 防抖
- **组合筛选**：以上四种条件可同时生效

---

### 4.5 学习看板

- **统计卡片**：总收藏数 / 待学习 / 学习中 / 已学会
- **进度图表**：各状态占比圆环图、每周完成趋势折线图、分类分布柱状图
- **学习日历**：月历视图，标记每日完成情况；连续学习天数（Streak）统计
- **最近动态**：最近 10 条状态变更时间轴

> 趋势图、学习日历、Streak、最近动态均依赖 `activity_logs` 表的历史记录。每次条目状态发生变更时，后端自动写入一条日志。

---

### 4.6 数据导入/导出

- **导出**：全量数据（分类、资源类型、条目、标签）导出为 JSON
- **导入**：解析同格式 JSON，支持合并或覆盖
- **书签导入**：支持从浏览器导出的书签 HTML 批量导入，自动解析 URL 和标题

## 6. 界面设计规格

### 色彩系统

| 角色 | 色值 | 用途 |
|------|------|------|
| 主色 | #C17B5C（暖棕） | 品牌色、CTA 按钮 |
| 辅色 | #F4A261（柔橙） | 高亮、待学习状态 |
| 背景 | #FFF9F5（米白） | 页面主背景 |
| 成功色 | #52B788（薄荷绿） | 已学会状态 |
| 文字 | #4A3728（深棕） | 正文内容 |
| 分割线 | #F0D5C0（淡肤） | 卡片边框 |

### 布局

- **左侧栏**（240px）：分类树形导航 + 底部管理入口
- **主内容区**（自适应）：筛选 Chips + 卡片网格/列表切换
- **顶部**：状态 Tab + 搜索框 + 添加按钮
- **响应式**：桌面 3 列 → 平板 2 列 → 手机单列

### 动效

- 卡片悬停：上移 3px + 阴影增强，200ms
- 删除动画：淡出 + 高度收起，250ms
- 弹窗出现：缩放 + 位移，300ms

## 7. 非功能性需求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏 < 2s；1000 条列表渲染流畅 |
| 可靠性 | 操作失败自动回滚 UI 并提示 |
| 安全性 | Supabase service_role key 后端鉴权，单用户数据隔离 |
| 可扩展性 | 分类和资源类型完全动态配置，无硬编码 |
| 浏览器兼容 | Chrome 100+ / Edge 100+ / Safari 15+ / Firefox 100+ |
